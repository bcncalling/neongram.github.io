Overview

mtproto-lib is a modern, Python-based library for interacting with the Telegram MTProto API. It provides a robust framework for building Telegram clients, featuring components for RPC handling, exception management, emoji support, enumerated types, filters, and handlers. The library is designed to be modular, extensible, and developer-friendly, with a focus on type safety and asynchronous programming.

Modules

__version__.py

Defines the version of the library, following semantic versioning.

Attributes

  • __version__ (str): The version string (e.g., "1.0.0").
  • VERSION (str): Alias for __version__.
  • VERSION_INFO (tuple[int]): Tuple of version numbers (e.g., (1, 0, 0)).

Example

from mtproto.__version__ import __version__, VERSION_INFO
print(f"Version: {__version__}")  # Version: 1.0.0
print(f"Version Info: {VERSION_INFO}")  # Version Info: (1, 0, 0)

Examples

Combined Filters and Handlers

from mtproto.utils.filters import Command, Text, create_and_filter
from mtproto.utils.handlers import HandlerManager, MessageHandler
from mtproto.client.neon_client import NeonClient
from mtproto.schema.tl_object import TLObject

async def handle_message(obj: TLObject, client: NeonClient):
    print(f"Message: {obj.values['message']}")

async def main():
    client = NeonClient(name="test", api_id=12345, api_hash="your_api_hash")
    manager = HandlerManager(client)
    filter = create_and_filter(Command("start"), Text("hello"))
    manager.add_handler(MessageHandler(handle_message, filter))

    # Simulate update
    class MockUpdate(TLObject):
        def __init__(self):
            self.values = {"message": "/start hello", "chat_id": 123}

    await manager.process_update(MockUpdate())

asyncio.run(main())
# Output: Message: /start hello
